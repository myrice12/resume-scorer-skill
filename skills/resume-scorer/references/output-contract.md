# 输出约定

推荐输出结构如下：

1. 先给一句结论，并包含总分；如果有 JD 对标，再带上 `fit score`。
2. 给出维度分表格，或平铺的分项得分列表。
3. 列出主要优势。
4. 列出主要缺口。
5. 给出按优先级排序的修改建议。

## 改写边界

- 保留用户原始事实。
- 如果原文缺少量化指标，应该提示“哪里需要补指标”，而不是直接编造数字。
- 如果原文缺少工具、职责范围或规模信息，改写前应提示用户确认，而不是擅自加入。
- 改写 bullet 时，优先采用 `动作 + 背景 + 方法 + 结果` 的结构。

## JD 对标说明

当简历或项目经历与 JD 对标时：

- 明确指出已匹配的关键词
- 列出价值最高的缺失关键词
- 说明这些词适合自然补到原文的哪个位置
